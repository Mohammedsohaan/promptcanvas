# PROJECT_RULES.md

## PromptCanvas Project Rules

This document contains the permanent rules for PromptCanvas.

These rules should not be changed unless the project owner explicitly approves the change.

---

## Project Information

Project Name: PromptCanvas

Tagline:
From Idea to Product Blueprint.

Current Version:
v0.1.0

Status:
Planning

---

## Goal

PromptCanvas helps people plan software products before they start coding.

The application focuses on turning ideas into structured product blueprints.

---

## Documentation Style

Always write documentation in a clean startup style.

Rules:

- Write in simple English.
- Use short paragraphs.
- Prefer bullet points.
- Avoid marketing language.
- Avoid AI-generated sounding text.
- Keep documentation easy to scan.

---

## Development Rules

- Build one feature at a time.
- Never skip planning.
- Never build features that are not part of the current version.
- Finish Version 1 before planning Version 2.

---

## Coding Rules

- Keep components small.
- Keep folder names simple.
- Use meaningful file names.
- Reuse components whenever possible.
- Don't add unnecessary libraries.

---

## Git Rules

Every work session ends with:

git status
git add .
git commit -m "Meaningful commit message"
git push

Commit messages should describe real progress.

Examples:

- Add authentication page
- Create dashboard layout
- Build product wizard
- Improve landing page
- Fix responsive navigation

## AI Rules

AI is an assistant.

The final decisions belong to the project owner.

Never generate large sections of the application without approval.

---

## Mentor Rules

The mentor must:

- Give only one step at a time.
- Explain why the step matters.
- Never change the roadmap without discussion.
- Never change the documentation style without approval.
- Never skip phases.

---

## Founder Rules

The founder will:

- Complete one step before moving to the next.
- Ask questions whenever something is unclear.
- Focus on quality over speed.
- Finish products before starting new ones.

---

## Long-Term Goal

Build 30–40 professional, deployed products before graduation.

Every project should be portfolio-ready and demonstrate real engineering practices.
