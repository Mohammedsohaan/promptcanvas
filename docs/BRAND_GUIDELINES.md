# PromptCanvas Brand Guidelines

## Philosophy

PromptCanvas is a software planning platform, not a code editor, project management tool, or conversational chatbot. Its primary purpose is to provide an environment for structured thought. Every design decision—from layout down to typography—must encourage clarity, focus, and rigorous planning before any development begins.

## Design Principles

- **Simplicity First:** Interfaces should surface only what is immediately necessary. Extraneous elements distract from the core task of architectural planning.
- **Calm Interface:** The environment should feel quiet and unhurried. We achieve this through generous whitespace, muted colors, and the elimination of visual clutter.
- **Professional Appearance:** The visual identity must inspire trust and technical competence. It should feel like a serious engineering tool.
- **Purposeful Motion:** Animation is exclusively a tool for communication, not decoration. Motion should explain state changes or illustrate workflows.
- **Consistency:** Predictable design patterns reduce cognitive load, allowing users to focus entirely on their software architecture rather than navigating the interface.

## Color System

The PromptCanvas palette is highly restrained to maintain a calm, focused environment.

- **Background:** Deep, immersive dark tones that reduce eye strain during long planning sessions.
- **Surface:** Slightly elevated from the background to distinguish interactive or grouped areas without sharp contrast.
- **Border:** Borders should be nearly imperceptible, serving only to define structure subtly.
- **Primary Text:** High contrast for optimal readability on dark backgrounds.
- **Secondary Text:** Muted tones used for supporting copy, timestamps, and metadata to establish a clear visual hierarchy.
- **Accent:** Used extremely sparingly. When applied, accents indicate active states, critical calls to action, or success metrics.

## Typography

- **Primary Font:** A highly legible, modern sans-serif. Chosen for its neutral tone and excellent readability at both small UI sizes and large display weights.
- **Heading Weight:** Bold with tight tracking. Headings should feel structural, solid, and deliberate.
- **Body Text:** Medium weight with a relaxed line-height. Ensures long-form documentation and specifications remain comfortable to read.
- **Monospace Font:** A clean, developer-focused monospace for code snippets, database schemas, and technical variables to reinforce the engineering focus of the platform.

## Layout & Spacing

- **Grid Philosophy:** We rely on rigid, predictable grid structures to organize complex information logically.
- **Section Spacing:** Generous padding between major sections to let content breathe and prevent overwhelming the user.
- **Component Spacing:** Logical clustering using strict gap scales to group related inputs and actions.
- **Card Spacing:** Internal card padding should be comfortable to frame content elegantly without feeling cramped.
- **Responsive Behavior:** The layout must gracefully scale down, prioritizing reading flow and structural integrity over cramming components side-by-side on smaller screens.

## Border Radius

Consistent border radii contribute to the premium feel of the interface:

- **Cards:** Soft enough to feel modern, structural enough to contain complex data.
- **Inputs:** Standard, predictable bounds for user text entry.
- **Buttons:** Standard rounding for general actions, or fully rounded for prominent, primary calls to action.
- **Modals:** Distinctly separated floating surfaces from the background layer.

## Motion Principles

Motion in PromptCanvas must always communicate product meaning and state. Valid use cases include:

- workflow progression
- blueprint generation
- page transitions
- loading states

**Strictly Avoid:** Decorative animations, excessive movement, cursor-follow effects, floating particles, and unnecessary visual noise. If an animation does not explain how the product works, it should be removed.

## Component Philosophy

- **Buttons:** Clear, predictable, and distinctly interactive. Primary buttons carry weight; secondary buttons recede into the background.
- **Cards:** Used strictly as containers for discrete concepts or planning modules. They should have subtle borders and extremely soft, low-opacity drop shadows.
- **Navigation:** Unobtrusive and persistent. It should guide without demanding attention.
- **Forms:** Clean, generously spaced, and forgiving. Inputs should have subtle active states.
- **Badges:** Small, highly legible markers for status or categorization, utilizing low-opacity background fills.
- **Dialogs:** Focused overlays for specific, isolated tasks that require the user's undivided attention.

## Product Identity

The visual identity of PromptCanvas is built to feel:

- premium
- modern
- minimal
- calm
- professional
- engineering-focused

Every pixel, shadow, and interaction must align with a single underlying truth: **PromptCanvas exists to help people think before they build.**
